ℚℝ 2 Conjunto de Dados

Um conjunto de dados é uma coleção estruturada de informações organizadas em linhas e colunas, onde cada linha representa uma observação e cada coluna representa uma variável. Os dados devem ser convenientes para:

  1. Análise: Facilitar a análise estatística e identificação de padrões.
  2. Modelagem: Servir como base para desenvolver e testar modelos preditivos.
  3. Decisão: Apoiar a tomada de decisões informadas em diversos contextos.
  4. Relatórios: Permitir a criação de relatórios e a comunicação de insights.
  5. Armazenamento: Dados devem ser organizados e armazenados para fácil acesso.

Conjuntos de dados são fundamentais para pesquisa, negócios, e tomada de decisões baseadas em evidências.

2.1 Organização e Notação

O conjunto de dados é organizado em uma estrutura tabular, onde os dados são dispostos em linhas e colunas. A matriz que representa o conjunto de dados é sempre denotada por uma letra maiúscula em negrito, por exemplo, \(\mathbf{X}\).

Uma matriz \(\mathbf{X}\) que representa o conjunto de dados pode ser formalmente descrita da seguinte maneira:

\[ \mathbf{X} = \begin{bmatrix} x_{11} & x_{12} & \cdots & x_{1j} \\ x_{21} & x_{22} & \cdots & x_{2j} \\ \vdots & \vdots & \ddots & \vdots \\ x_{i1} & x_{i2} & \cdots & x_{ij} \end{bmatrix} \]

onde:

  • \(\mathbf{X}\) é a matriz que representa o conjunto de dados.
  • \(x_{ij}\): é o elemento localizado na i-ésima linha e j-ésima coluna da matriz \(\mathbf{X}\). Isto é, representa o valor da j-ésima variável para o i-ésimo objeto.
  • Linhas: Cada linha i corresponde a um objeto individual, e todos os valores na linha i representam as medições ou características desse objeto em relação a todas as variáveis. Também é possível interpretar i como sendo o número total de objetos (amostras) no conjunto de dados.
  • Colunas: Cada coluna j representa uma variável específica, e todos os valores na coluna j representam as medições dessa variável em todos os objetos. Também é possível interpretar j como sendo o número total de variáveis (dimensões) medidas.
  • Notações variantes, mais compactas e menos formais, também são comumentes encontrada na literatura como: \(\mathbf{X}(I,J)\). Aqui, temos \(I\) linhas e \(J\) colunas na matriz \(\mathbf{X}\).

Exemplificando…

Se tivermos um conjunto de dados com 3 objetos e 4 variáveis, a matriz \(\mathbf{X}\) pode ser representada como:

\[ \mathbf{X} = \begin{bmatrix} x_{11} & x_{12} & x_{13} & x_{14} \\ x_{21} & x_{22} & x_{23} & x_{24} \\ x_{31} & x_{32} & x_{33} & x_{34} \end{bmatrix} \]

onde \(x_{ij}\) representa o valor da j-ésima variável para o i-ésimo objeto.

Para as 3 primeiras linhas das 4 variáveis do cojunto de dados Iris, temos a matriz:

\[ \mathbf{X} = \begin{bmatrix} 5.10 & 3.50 & 1.40 & 0.20 \\ 4.90 & 3.00 & 1.40 & 0.20 \\ 4.70 & 3.20 & 1.30 & 0.20 \\ \end{bmatrix} \]

 

Aproveitando para Explicar data.frame e matrix no R!

No R, os conjuntos de dados ou matrizes de dados são comumente representados por data frames (uma estrutura de dados computacional que muitas linguagens modernas também adotaram). Os data frames são estruturas de dados muito convenientes, pois suportam diferentes tipos de variáveis. Isso contrasta com as matrizes, que são restritas a um único tipo de dado. Ambas estruturas têm a capacidade de armazenar nomes de variáveis (colunas) e nomes de amostras (linhas) no R. A percepção de um data.frame é de uma estrutura muito mais semelhante com a de uma tabela convencional, como ilustrado a seguir:

Table 2.1: Parte do dataset Iris
Sepal.Length Sepal.Width Petal.Length Petal.Width
1 5.1 3.5 1.4 0.2
2 4.9 3.0 1.4 0.2
3 4.7 3.2 1.3 0.2

Para maior clareza, a seguir apresentamos a tabela implementada usando duas estruturas diferentes: data.frame e matrix. Visualmente, ambas as estruturas são indistinguíveis.

> X <- head( iris[,-5], 3 )
> X
  Sepal.Length Sepal.Width Petal.Length Petal.Width
1          5.1         3.5          1.4         0.2
2          4.9         3.0          1.4         0.2
3          4.7         3.2          1.3         0.2
> class( X )
[1] "data.frame"
> Y <- as.matrix( X )
> Y
  Sepal.Length Sepal.Width Petal.Length Petal.Width
1          5.1         3.5          1.4         0.2
2          4.9         3.0          1.4         0.2
3          4.7         3.2          1.3         0.2
> class( Y )
[1] "matrix" "array" 
>

Aqui, X é um objeto da classe data.frame e Y é um objeto da classe matrix. A seguir, fornecemos uma explicação para cada comando.

  • O comando X <- head( iris[,-5], 3 ) atribui a X(3,4) parte de iris (iris é 150x5).
  • Digitar somente X, imprime seus valores na tela.
  • O comando class(X) mostra que a classe do objeto X é data.frame.
  • Y <- as.matrix( X ) cria um objeto Y a partir de X e o força a ser da classe matrix
  • Digitar somente Y, imprime seus valores na tela.
  • O comando class(Y) foi somente para confirmar que Y é uma matriz.

 

Algumas propriedades da álgebra matricial são aplicáveis a ambas as classes. No entanto, quando há potencial para ambiguidades, é necessário realizar conversões explícitas entre as classes (como feito anteriormente).

Além disso, é importante estar atento a conversões implícitas de tipos (ou classes), que podem ocorrer automaticamente. O exemplo a seguir ilustra esse comportamento:

> X <- head( iris[,-5], 3 )
> class(X)
[1] "data.frame"
> Y <- t( X )
> class(Y)
[1] "matrix" "array"

A operação de transposição em X é permitida, mas o resultado é um objeto da classe matrix. Esteja atento a esses detalhes.

  • O comando t(X) recebe uma matriz ou um data.frame X como argumento e retorna a transposta de X

2.2 Valores faltantes: NA

No R, NA, sigla de Not Available, é usado para representar valores faltantes em um conjunto de dados.

A indicação é necessária para que o analista decida se haverá imputação de dados ou se haverá remoção das amostras que contenha parte dos dados perdidos.

Aqui estão algumas maneiras comuns de lidar com NA no R:

  1. Identificação de NA

Você pode identificar NA usando a função is.na().

x <- c(1, 2, NA, 4, NA)
is.na(x) # Verifica posições onde há algum NA

# Alternativa 
any(is.na(x))  # Verifica se há algum NA
  1. Remoção de NA

Para remover NA de um vetor ou dataframe, você pode usar a função na.omit() ou o argumento na.rm em funções como mean(), sum(), etc.

Para vetores:

x <- c(1, 2, NA, 4, NA)
x_clean <- na.omit(x)

Para dataframes:

df <- data.frame(a = c(1, 2, NA), b = c(NA, 3, 4))
df_clean <- na.omit(df)
  1. Substituindo NA

Substituir NA por um valor específico.

x <- c(1, 2, NA, 4, NA)
x[is.na(x)] <- 0  # Substituindo NA por 0
  1. Análise Estatística com NA

Algumas funções permitem que você especifique como lidar com NA usando o argumento na.rm.

x <- c(1, 2, NA, 4)
mean(x, na.rm = TRUE)  # Calcula a média ignorando NA

Imputação

Imputação de dados é uma técnica utilizada na Estatística e em Ciência de Dados para lidar com dados faltantes ou ausentes em um conjunto de dados. Em vez de simplesmente excluir registros com dados faltantes, a imputação visa preencher esses valores ausentes com estimativas baseadas em informações disponíveis no conjunto de dados. O objetivo é melhorar a qualidade dos dados e permitir análises mais precisas e completas. Entre as principais técnicas estão:

  • Imputação Simples: Substitui valores faltantes por uma constante, como a média ou mediana dos valores existentes.

  • Imputação por Regressão: Usa um modelo de regressão para prever e preencher valores faltantes com base em variáveis preditoras.

  • Imputação por Métodos de Machine Learning: Emprega algoritmos como K-Nearest Neighbors (KNN) ou Random Forests para prever valores faltantes com base em padrões encontrados nos dados.

  • Imputação por Métodos Estatísticos: Inclui técnicas como múltiplas imputações, que criam vários conjuntos de dados imputados e combinam os resultados para uma análise mais robusta.

Considerações

  • Qualidade dos Dados: A escolha da técnica de imputação depende da natureza dos dados e do mecanismo de ausência. Técnicas mais complexas podem ser necessárias para conjuntos de dados grandes ou para variáveis que têm relações não lineares.

  • Impacto na Análise: A imputação pode introduzir vieses ou variabilidade adicional nos dados, especialmente se as estimativas imputadas não forem precisas.

A imputação de dados é uma técnica importante para preparar os dados para análise, e a escolha da técnica deve levar em conta a estrutura dos dados e o objetivo da análise. No entanto, neste caderno, não vamos abordar a imputação de dados.

2.3 Dados Disponíveis

Para aprender e praticar análise de dados, é essencial ter acesso a conjuntos de dados variados. No R, o pacote datasets oferece uma coleção de dados embutidos que vem com a instalação padrão do software. Esse pacote abrange uma ampla gama de conjuntos de dados, cobrindo diversas áreas como biologia, economia e ciências sociais. Com esses dados, os usuários podem praticar técnicas de análise, visualização e modelagem, facilitando o aprendizado prático e a aplicação de conceitos estatísticos. Para visualizar a lista completa de conjuntos de dados disponíveis, use o comando: help( package='datasets' ).

Para carregar um conjunto de dados em sua sessão, utilize o comando data() seguido do nome do conjunto. Por exemplo: data( 'iris' ).

Além disso, a maioria dos pacotes no R inclui seus próprios conjuntos de dados. Para acessá-los, basta usar o comando data( 'nome_do_dataset', package='nome_do_pacote' ) após a instalação do pacote.”

    • Fourier transform and near infrared dataset of dialdehyde celluloses used to determine the degree of oxidation with chemometric analysis

    Jonas Simon, Otgontuul Tsetsgee, Nohman Arshad Iqbal, Janak Sapkota, Matti Ristolainen, Thomas Rosenau, Antje Potthast,
    Data in Brief, Volume 40, 2022,
    https://doi.org/10.1016/j.dib.2021.107757.
    Este conjunto de dados está relacionado ao artigo de pesquisa intitulado: A fast method to measure the degree of oxidation of dialdehyde celluloses using multivariate calibration and infrared spectroscopy. No artigo, 74 amostras de dialdeído celulose com diferentes graus de oxidação foram preparadas por oxidação de periodato e analisadas por espectroscopia de infravermelho com transformada de Fourier (FTIR) e infravermelho próximo (NIR). Os modelos NIR/PLSR e FTIR/PLSR desenvolvidos podem ser facilmente implementados em outros laboratórios para prever de forma rápida e confiável o grau de oxidação das celuloses de dialdeído.
