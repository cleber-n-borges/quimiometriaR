ℚℝ 4 Pré-Processamento de Variáveis

4.1 Autoescalamento

O autoescalamento é um processo de pré-processamento de dados amplamente utilizado em estatística, aprendizado de máquina e quimiometria para transformar os dados para uma escala comum. Isso facilita a análise e modelagem, especialmente quando diferentes variáveis possuem magnitudes e unidades diferentes. O autoescalamento é fundamental para garantir que todas as variáveis contribuam igualmente para a análise, evitando que variáveis com escalas maiores dominem o modelo.

Esta transformação ajusta a escala dos dados e a centraliza em torno de zero, facilitando comparações e análises subsequentes.

Fórmula do Autoescalamento

Para uma variável \(x\) com média μ e desvio padrão σ, o autoescalamento é realizado pela transformação:

 

\[ x_{autoescalado} = (x − μ ) / σ \]

Onde:

  • \(x\) representa os valores da variável original.
  • μ é a média da variável.
  • σ é o desvio padrão da variável.
  • \(x_{autoescalado}\) são os valores transformados.

 

Propriedades do Autoescalamento

  1. Uniformização da Escala: Garante que todas as variáveis estejam na mesma escala, o que é especialmente útil em algoritmos que são sensíveis à escala dos dados, como algoritmos de clustering (e.g., K-means) e métodos de aprendizado de máquina (e.g., regressão linear e máquinas de vetor de suporte).

  2. Melhoria da Convergência: Pode acelerar a convergência de algoritmos de otimização que utilizam gradientes, pois elimina a influência desproporcional de variáveis com escalas maiores.

  3. Interpretação Consistente: Facilita a interpretação dos coeficientes de modelos, especialmente em regressão, onde variáveis com diferentes escalas podem obscurecer a relação entre variáveis e respostas.

  4. Comparação e Visualização: Facilita a comparação entre variáveis e a visualização dos dados em gráficos, pois todas as variáveis têm a mesma escala. (Comumente, os softwares que criam gráficos autoescalam as variáveis para melhorar a clareza das dispersões dos pontos, mas mantêm os rótulos das variáveis originais. Esta é uma das aplicações do autoescalamento).

 

Aplicações Comuns

  • Aprendizado de Máquina: Antes de aplicar algoritmos de aprendizado de máquina, como K-means, PCA (Análise de Componentes Principais), e regressão, os dados são frequentemente autoescalados para garantir um desempenho eficaz e resultados confiáveis.

  • Análise Estatística: Em análises estatísticas multivariadas, como a Análise de Componentes Principais (PCA) e Análise Discriminante Linear (LDA), o autoescalamento é usado para garantir que cada variável contribua de forma equitativa para as análises.

  • Quimiometria: Na análise de dados espectroscópicos e quimiométricos, o autoescalamento ajuda a normalizar os dados para melhorar a interpretação dos modelos e a precisão das previsões.

 

Exemplo Prático em R

Abaixo segue um exemplo de como realizar o autoescalamento de dados em R, usando a função scale():

# Carregar pacotes necessários
library(ggplot2)

# Definir o seed para reprodutibilidade
set.seed(42)

# Criar um dataframe com 2 variáveis e 25 amostras
df <- data.frame(
  Variavel1 = rnorm(25, mean = 50, sd = 10),
  Variavel2 = rnorm(25, mean = 100, sd = 20)
)

# Exibir o dataframe original
print("Dataframe Original:")
print(df)

# Aplicar autoescalamento
df_autoescalado <- as.data.frame( scale(df) )

# Exibir o dataframe autoescalado
print("Dataframe Autoescalado:")
print(df_autoescalado)

# Gráfico dos dados originais
p1 <- ggplot(df, aes(x = Variavel1, y = Variavel2)) +
  geom_point(color = "blue", size=2) +
  labs(title = "Dados Originais", x = "Variável 1", y = "Variável 2")

# Gráfico dos dados autoescalados
p2 <- ggplot(df_autoescalado, aes(x = Variavel1, y = Variavel2)) +
  geom_point(color = "red", size=2) +
  labs(title = "Dados Autoescalados",
    x = "Variável 1 (Autoescalado)", y = "Variável 2 (Autoescalado)")

# Mostrar os gráficos
library(gridExtra)
grid.arrange(p1, p2, ncol = 2)

Você pode copiar e colar esse código em um script R para gerar e visualizar os gráficos dos dados originais e autoescalados.

 

Explicação do Código

  1. Criação do Dataframe: Utilizamos rnorm() para gerar 25 amostras para duas variáveis com médias e desvios padrão diferentes.

  2. Autoescalamento: Aplicamos a função scale() para normalizar os dados, transformando-os para ter média zero e desvio padrão um.

  3. Gráficos:

  • Dados Originais: Exibimos um gráfico de dispersão dos dados originais.
  • Dados Autoescalados: Exibimos um gráfico de dispersão dos dados após o autoescalamento.
  1. Visualização: Usamos gridExtra para mostrar os gráficos lado a lado.

O autoescalamento é uma técnica essencial no pré-processamento de dados, garantindo que todos os dados sejam tratados de maneira justa e adequada para análises e modelagens subsequentes.

4.2 Normalização Min-Max:

Embora a normalização pelos valores mínimos e máximos não seja completamente resistente a outliers, ela pode reduzir o impacto que valores extremos têm na análise. Outliers podem distorcer a escala e a interpretação dos dados, e a normalização ajuda a mitigar esse efeito ao transformar os dados para um intervalo padrão.

 

Fórmula para a normalização

\[ x_{normalizado} = ( x − x_{min} ) / ( x_{max} - x_{min} ) \]

onde:

  • \(x\) representa os dados originais
  • \(x_{min}\) é o valor mínimo da variável
  • \(x_{min}\) é o valor máximo da variável
  • \(x_{normalizado}\) são os dados normalizados

 

Passos para Normalização Min-Max

  • Determine os Valores Mínimo e Máximo: Encontre os valores mínimo e máximo da variável que você deseja normalizar.

  • Aplicar a Fórmula: Para cada variável \(x\), aplique a fórmula para transformar o valor para o intervalo [0, 1].

 

Exemplo Prático em R

# Criar um vetor de dados
dados <- c(10, 20, 30, 40, 50)

# Normalizar os dados para o intervalo [0, 1]
normalizado <- (dados - min(dados)) / (max(dados) - min(dados))

# Exibir os dados normalizados
print(normalizado)

# Para normalizar os dados contidos em um dataframe ou uma matriz

# Definir o seed para reprodutibilidade
set.seed(42)

# Criar um dataframe com 2 variáveis e 25 amostras
df <- data.frame(
  Variavel1 = rnorm(25, mean = 50, sd = 10),
  Variavel2 = rnorm(25, mean = 100, sd = 20)
)

# Aplicar a normalização Min-Max
df_normalizado <- apply( df, 2, function(df)(df - min(df)) / (max(df) - min(df))  )

#Exibir os dados normalizados
print( df_normalizado)

 

Explicação do Trecho do Código

# Aplicar a normalização Min-Max
df_normalizado <- apply( df, 2, function(df)(df - min(df)) / (max(df) - min(df))  )

Passo a Passo:

  1. Função apply():
  • apply() é uma função em R que aplica uma função especificada a margens de uma matriz ou data frame.
  • A sintaxe básica é apply(X, MARGIN, FUN, ...), onde X é a matriz ou data frame, MARGIN indica a margem sobre a qual a função deve ser aplicada (1 para linhas e 2 para colunas), e FUN é a função a ser aplicada. (Estes detalhes estão no help da função apply).
  1. Argumentos do apply():
  • df: Este é o data frame ou matriz ao qual a função será aplicada.
  • 2: Este argumento indica que a função deve ser aplicada a cada coluna do data frame (MARGIN = 2 significa aplicar a função por coluna).
  • function(df)(df - min(df)) / (max(df) - min(df)): Este é o código da função que será aplicada a cada coluna.
  1. Função anônima:
  • function(df) (df - min(df)) / (max(df) - min(df)) é uma função anônima (ou seja, uma função definida sem nome).
  • Essa função realiza a normalização Min-Max para cada coluna do data frame. A normalização Min-Max transforma os valores para uma faixa de 0 a 1.

Destrinchando a Função Anônima:

  1. df - min(df):
  • Para cada valor na coluna, subtrai o valor mínimo dessa coluna. Isso é feito para ajustar os valores, começando do zero.
  1. max(df) - min(df):
  • Calcula a diferença entre o valor máximo e o valor mínimo da coluna (também chamado de amplitude). Isso é usado para dimensionar os valores ajustados para a faixa de 0 a 1.
  1. (df - min(df)) / (max(df) - min(df)):
  • Divide o ajuste realizado na primeira etapa pela faixa (a diferença entre o valor máximo e o valor mínimo). O resultado é que os valores da coluna são escalados para o intervalo de 0 a 1.

Resultado:

  • df_normalizado:
    • O resultado do apply() é um novo data frame (ou matriz, dependendo da estrutura original de df) onde cada coluna é normalizada usando a técnica Min-Max. Os valores em cada coluna vão variar entre 0 e 1.

Considerações Adicionais:

  • Estrutura do Data Frame:
    • apply() pode transformar um data frame em uma matriz, o que pode causar problemas se o data frame original contiver tipos de dados mistos (por exemplo, números e caracteres). Se for o caso, considere converter para uma matriz ou usar funções específicas para data frames, como lapply().

4.3 Transformações

É muito comum, na construção de modelos preditivos, o uso de transformações40 de variáveis para abordar e corrigir vários problemas corriqueiros como assimetria na distribuição, heterocedasticidade, não linearidade e outliers. O uso adequado dessas transformações pode melhorar significativamente a eficácia das previsões, isto é, resultar em modelos mais precisos, robustos e interpretáveis. As transformações têm como objetivo abordar os seguintes aspectos:

1. Correção da Assimetria: Dados assimétricos (ou seja, que apresentam uma cauda mais longa de um lado do que do outro) podem levar a ajustes imprecisos dos modelos prejudicando o desempenho preditivo, pois muitos modelos estatísticos assumem que os dados seguem uma distribuição normal. Transformações como a logarítmica, a raiz quadrada ou a Box-Cox podem ajudar a suavizar a distribuição dos dados, tornando-a mais simétrica e, assim, melhorando a precisão do modelo.

2. Estabilização da Variância: Em muitos casos, a variância dos dados não é constante em diferentes níveis da variável preditora, um fenômeno conhecido como heterocedasticidade. Isso pode levar a previsões imprecisas e inferências incorretas. Transformações como a Box-Cox ou a logarítmica podem estabilizar a variância, fazendo com que a variabilidade dos dados seja mais uniforme e adequada para modelos que assumem homocedasticidade.

3. Melhora na Linearidade: Muitos modelos preditivos, como a regressão linear, pressupõem uma relação linear entre as variáveis independentes e a variável dependente. No entanto, a relação real pode ser não linear. Transformações podem transformar essas relações não lineares em lineares, facilitando a modelagem e a interpretação dos dados. Por exemplo, uma transformação logarítmica pode transformar uma relação exponencial em uma relação linear, melhorando a adequação do modelo.

4. Redução do Impacto de Outliers: Transformações podem reduzir o impacto de valores extremos ou outliers, que podem distorcer a análise e a construção do modelo. Por exemplo, a transformação logarítmica pode minimizar o efeito de valores muito altos, tornando-os menos influentes no ajuste do modelo.

5. Melhoria na Interpretabilidade: Em alguns casos, a transformação das variáveis pode facilitar a interpretação dos coeficientes do modelo. Por exemplo, em uma transformação logarítmica, o coeficiente da variável pode ser interpretado como a taxa de crescimento percentual, o que pode ser mais intuitivo e informativo para decisões práticas.

6. Ajuste para Problemas Específicos de Dados: Alguns tipos de dados, como contagens ou dados que variam em ordens de magnitude, podem se beneficiar de transformações específicas que são mais adequadas para o tipo de dados em questão. Transformações apropriadas podem levar a modelos mais precisos e robustos.

4.3.1 Logarítmica e Raiz Quadrada

As transformações por aplicação da operação logarítmica e da operação raiz quadrada são amplamente empregadas quando a variável apresenta uma distribuição assimétrica ou uma variância elevada. São monotônicas, o que significa que preserva a ordem dos dados. Isso é crucial para modelos preditivos, pois assegura que a transformação não altere a capacidade do modelo de capturar a relação entre as variáveis. Também são não-paramétricas.

 

Transformação logarítmica:

\[ y_{transformada} = log( y ) \]

Transformação raiz quadrada:

\[ y_{transformada} = \sqrt y \]

onde:

  • \(y\) é a variável original a ser transformada.
  • \(y_{transformada}\) é a variável transformada.

 

A diferença básica é entre as duas operações é o quão rapidamente conseguem contrair a escala original. Graficamente é mais intuitivo essa percepção.

4.3.2 Transformação Box-Cox

A Transformação Box-Cox41 é uma técnica estatística utilizada para estabilizar a variância e tornar os dados mais próximos de uma distribuição normal, o que pode ser útil para melhorar a qualidade de modelos estatísticos e de machine learning.

Aqui estão alguns pontos principais sobre a Transformação Box-Cox:

  1. Objetivo:

A transformação visa lidar com problemas de heterocedasticidade (variação não constante) e não-normalidade nos dados. A normalidade e a homocedasticidade são frequentemente pressupostos em muitos testes estatísticos e modelos de regressão.

  1. Formulação Matemática:

A fórmula da Transformação Box-Cox é dada por:

\[ y(\lambda) = \begin{cases} \frac{y^\lambda - 1}{\lambda} & \text{se } \lambda \neq 0 \\ \log(y) & \text{se } \lambda = 0 \end{cases} \]

onde:

  • \(y\) é a variável original a ser transformada.
  • \(\lambda\) é o parâmetro de transformação, que pode assumir qualquer valor real.
  1. Escolha de \(\lambda\):

O parâmetro \(\lambda\) é ajustado para encontrar a melhor transformação possível dos dados. A escolha de \(\lambda\) é geralmente feita através de métodos de da maximização da função de verossimilhança, que busca o valor \(\lambda\) que melhor estabiliza a variância maximiza e melhor proximidade dos dados transformados a uma distribuição normal. Em alguns softwares estatísticos, você pode encontrar o melhor \(\lambda\) automaticamente. A transformação ajuda a reduzir a heterocedasticidade, ou seja, a variação não constante dos resíduos em um modelo de regressão. O melhor ajuste dos dados a uma distribuição normal pode melhorar a adequação dos modelos estatísticos e facilitar a interpretação dos resultados. A transformação é uma forma generalizada de várias outras transformações comuns, como a transformação logarítmica (para \(\lambda = 0\)) e a raiz quadrada (\(\lambda = 0.5\)).

  1. Restrições:

A Transformação Box-Cox só pode ser aplicada a dados positivos (ou seja, \(y > 0\)). Se seus dados contiverem valores não positivos, pode ser necessário adicionar uma constante aos dados antes da transformação ou utilizar outras transformações. A escolha de \(\lambda\) pode ser sensível e pode variar dependendo dos dados. Uma transformação que funciona bem para um conjunto de dados pode não ser ideal para outro (Existem outras transformações, como a transformação logarítmica simples ou a transformação de raiz quadrada, que podem ser mais apropriadas dependendo do contexto dos dados e dos objetivos da análise).

  1. Aplicações:

A Transformação Box-Cox é amplamente utilizada em análise de regressão, ANOVA e outros modelos estatísticos onde os pressupostos de normalidade e homocedasticidade são importantes. Ela pode melhorar a validade dos resultados e a eficácia dos modelos.

  • Modelagem Estatística: Em modelos de regressão e ANOVA, onde pressupostos de normalidade e homocedasticidade são importantes, a Transformação Box-Cox pode melhorar o ajuste do modelo e a validade dos testes.

  • Previsão: Em modelos de séries temporais e de previsão, a transformação pode estabilizar a variância das observações ao longo do tempo, tornando os modelos mais robustos.

  • Análise Exploratória: A transformação pode ser usada para explorar a natureza dos dados e identificar a melhor abordagem para análise e modelagem.

  1. Interpretação:

Após aplicar a transformação, interpretar os resultados pode ser menos intuitivo exigir cuidados, pois a transformação altera a escala dos dados. Dependendo de \(\lambda\), a interpretação dos coeficientes e das previsões pode exigir uma adaptação. Em casos particulares, pode ser necessário traduzir os resultados transformados de volta para a escala original.

Em síntese, a Transformação Box-Cox é utilizada para ajustar dados e melhorar a performance dos modelos estatísticos, ajudando a lidar com problemas comuns como heterocedasticidade e não-normalidade. No entanto, como qualquer ferramenta estatística, deve ser usada com compreensão de suas limitações e com cuidado na interpretação dos resultados.

 

No R, o pacote MASS possui a função boxcox que implementa essa transformação.

4.3.3 Logística e Logit

Função Logística

A função logística42 é a curva com um formato de “S” quando representada graficamente, sendo conhecidad como sigmoidal. O contradomínio da função começa em \(0\), sobe rapidamente, e depois se aproxima de \(1\), criando uma transição suave entre esses dois limites. É definida pela fórmula:

\[ z = \left(\frac{1}{1+e^{-y}}\right) \]

À medida que \(y\) tende a \(-\infty\), a função se aproxima de \(0\). À medida que \(y\) tende a \(\infty\), a função se aproxima de \(1\). O ponto de inflexão da função logística ocorre em \(y = 0\), onde a função assume o valor \(0,5\). Esse ponto é onde a função muda de crescer lentamente para crescer rapidamente. A função pode ser generalizada facilmente para contemplar os seus ajustes a outros limites.

Uso da Função Logística

  • Modelagem de Probabilidades: A função logística é usada para mapear qualquer valor real \(z\) para um intervalo entre \(0\) e \(1\), tornando-a útil para modelar probabilidades. Isso é especialmente importante em modelos de regressão logística, onde a saída do modelo deve ser uma probabilidade.

  • Modelos de Crescimento: Em biologia e outras ciências, a função logística é usada para descrever o crescimento populacional limitado por fatores de capacidade de carga.

  • Redes Neurais: Na aprendizagem de máquinas e redes neurais, a função logística pode ser usada como uma função de ativação para introduzir não-linearidade nos modelos.

Relação com a Função Logit

A função logística é a inversa da função logit. Enquanto a função logit transforma uma probabilidade em uma escala não restrita, a função logística faz a transformação inversa, convertendo um valor em uma probabilidade. Se você tem um valor \(z\) (que pode ser o resultado de uma combinação linear de variáveis independentes em um modelo de regressão logística), a função logística é usada para obter a probabilidade \(p\). Dessa forma, a função logística é essencial para transformar as previsões de um modelo de regressão logística em probabilidades compreensíveis e interpretáveis.

Função logit

A função logit43 é uma função matemática que é frequentemente usada em modelos estatísticos, especialmente na regressão logística. A função logit é definida como o logaritmo da razão de chances (odds) e é dada pela fórmula:

\[ \text{logit}(p) = \log \left(\frac{p}{1-p}\right) \]

onde \(p\) é a probabilidade de um evento ocorrer, e \(\frac{p}{1-p}\) é a razão de chances do evento.

A função logit transforma uma probabilidade \(p\) (que varia entre \(0\) e \(1\)) em uma escala que vai de \(-\infty\) a \(\infty\).

Isso é útil em modelos de regressão logística porque a probabilidade \(p\) é restrita ao intervalo \([0,1]\), mas a função logit permite que a relação entre variáveis preditoras e a probabilidade de um evento seja modelada em uma escala não restrita.

A função logit é a função inversa da função logística.
